# 1. Js是一门动态类型的语言（.语法）

# 2. 弱类型

  • 数据类型划分不严格 var 声明

  • String int float double boolean null object

# 1. 是一门基于对象的语言

  • 封装、继承、多态、类class

# 2. ECMAScript：基础语法、DOM：文档对象模型、BOM：浏览器对象模型

# 3. 异步加载 async defor 下载Js的同时解析HTML

# 4. 是一门解释型的语言

  • 同一个script标签中，前一个报错会影响后面的执行

8.Var a = 3;
  Var b = 4;
  将ab变量的值互换方法
（1）创建一个空变量c，将a和b互换；
（2）a = a+b //a = 7;
     b = a-b //b = 7-4 = 3;
  a = a-b //a =7-3 = 4;
（3）使用异或
    a = a ^ b;
    b = a ^ b; // b = a^b^b
  a = a ^ b; // a = a^b^a
  先将a和b的数值转为二进制
  除奇取余，直到商为零，余数反转。
  3的二进制是011
  4的二进制是100

  判断个个位的值如果相同则为0，如果不同则为1；
  011和100判断完为111；
  将111转为十进制
  底数：111
  基数：当前为多少进制就是对应的基数
  权次幂：从右往左编号，从零开始，依次加一
  结果：底数*基数的权次幂
  结果 = 1*2º+1*2¹+1*2² = 1+2+4 = 7
  所以a^b = 3^4 = 7

# 1. |和||的区别

  • 效率的区别：||的效率更高

  • a>b |c>d 左边为真（true）的时候继续判断右边

  • a>b ||c>d 左边为真（true）则直接返回真（true）

# 2. &和&&的区别

  • 效率的区别：&&的效率更高

  • a>b &c>d 左边为假（false）的时候继续判断右边

  • a>b &&c>d 左边为假（false）则直接返回假（false）

# 3. null有时候也会被当做对象类型，这就是这门语言的BUG

# 4. script标签放在哪最合适？

  • 不适合放在头部

  • 不适合放在html下面

  • Async和defer实现异步加载

  • 暂时放在body的最后面

  • 外部引用时，script标签里不能写任何代码（不会报错，但是没有效果）；

# 1. 严格模式（在script标签开始时加入”use strict”即进入严格模式）

在严格模式下this指向不能指向window，如果指向window则会返回undefined 但是仍然可以用call apply bind来改变this的指向 严格模式的优点： 消除代码运行的一些不安全之处，保证代码运行的安全； 提高编译器效率，增加运行速度； 为未来新版本的JavaScript做好铺垫。 严格模式的限制：

  • （1）不允许使用未声明的变量（例如x = 3使用之前必须要用var声明）
  • （2）不允许删除变量或对象。（delete x会报错）
  • （3）不允许删除函数。（function x(a,b){} delete x会报错）
  • （4）不允许变量重名。（function x(a,a){} 报错）
  • （5）不允许使用八进制。（var x = 010; 报错）
  • （6）不允许使用转义字符。（var x = /010;报错）
  • （7）不允许对只读属性赋值。
  • （8）不允许对一个使用getter方法读取的属性进行赋值
  • （9）不允许删除一个不允许删除的属性。（delete Object.prototype; 报错）
  • （10）变量名不能使用 "eval" 字符串。
  • （11）变量名不能使用 "arguments" 字符串。
  • （12）不允许使用这种语句：with (Math){x = cos(2)};
  • （13）由于一些安全原因，在作用域 eval() 创建的变量不能被调用
  • （14）禁止this关键字指向全局对象。

严格模式保留了一些关键字： Implements interface let package private protected public static yield
